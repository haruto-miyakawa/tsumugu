# つむぐ — 機能仕様書 (FEATURES)

このドキュメントは `HANDOFF.md` の補足で、各画面・各機能の振る舞いを実装可能な粒度で書き下ろしたものです。プロトタイプ (`index.html` 以下) と対応します。

---

## 0. グローバル要件

### 0.1 認証
- メール + パスワード / Google / X (Twitter) でサインイン
- 未ログインでも `/preview/[shareId]` だけは閲覧可

### 0.2 ナビゲーション
- 上部固定タブ: **ホーム / エディタ / ライブラリ / 設定**
- 右上: 検索（⌘+K）、通知ベル、アバター
- モバイルは下部タブバー + 中央に大きな「+」FAB（新規作成）

### 0.3 自動保存
- 全てのエディタ入力はデバウンス 800ms で自動保存
- 保存タイムスタンプを上部に表示（例: `自動保存 14:22`）
- オフラインキューイング（IndexedDB）、復帰時に同期

### 0.4 ショートカット
| キー | 動作 |
|---|---|
| `⌘ + K` | コマンドパレット / 検索 |
| `⌘ + N` | 新規記事 |
| `⌘ + S` | 強制保存 |
| `⌘ + Enter` | AIで生成 |
| `Tab` | ゴーストテキスト accept |
| `Esc` | ゴーストテキスト reject / モーダル閉じ |
| `⌘ + /` | AIサイドバー トグル |
| `⌘ + .` | 集中モード トグル |

## 1. ダッシュボード (ホーム)

### 1.1 つむぎの挨拶
- 時刻帯で挨拶文を出し分け（GOOD MORNING / AFTERNOON / EVENING）
- ユーザー名を埋め込み（「KOTONO」の位置）
- マスコット表情: 起動時 `happy`、操作中 `idle`、AI処理中 `thinking`、生成完了 `wink`

### 1.2 統計タイル (右上カード3枚)
| 項目 | 内容 | 更新タイミング |
|---|---|---|
| **STREAK** | 連続執筆日数 (1記事 ≥ 100字書いた日をカウント) | 毎日 0:00 JST |
| **PUBLISHED** | 公開済記事の累計 | 公開時 |
| **WORDS / WK** | 直近7日間の累計文字数 + WoW差分 | リアルタイム |

クリックで `/insights` (将来実装) へ遷移。

### 1.3 クイック生成パネル
- **入力**: テーマ一行 (最大100字、placeholder ローテーション)
- **設定チップ**: ジャンル / 文字数 / 口調（後述 §1.4）
- **Enter または「生成」ボタン**: `/api/draft` を叩き、`/editor/[id]` へ遷移しながらストリーミング開始
- 生成失敗時: マスコット表情 `thinking`、再試行ボタン

### 1.4 生成オプション (チップ)
- **ジャンル**: エッセイ / ハウツー / 体験記 / レビュー / ニュース解説 / コラム / 詩・短歌
- **文字数**: 500 / 1000 / 1500 / 2500 / 4000
- **口調**: です・ます / だ・である / カジュアル / 詩的 / 丁寧な手紙風
- **構造**: 起承転結 / 結論先出し / 三幕構成 / 散文 / Q&A
- **+ 設定**: モーダルで上記を細かく調整（保存可、プリセット化）

### 1.5 下書き一覧 (DRAFTS)
- フィルタチップ: **すべて / 下書き / 予約 / 公開済**
- 並び替え: 更新日時(default) / 作成日時 / 文字数 / 進捗率
- カード1件あたりの情報:
  - マスコットアバター（その記事のトーンを表す表情）
  - タイトル（クリックで `/editor/[id]`）
  - ジャンル · 文字数 · 状態 (AI下書き60%, 構成案, 仕上げ など)
  - 進捗率バー (0-100%)
  - 最終更新日時
  - 右端: `→` で開く、`⋯` でメニュー (複製 / アーカイブ / 削除 / 共有)
- 編集中の記事には左に4pxアクセントカラーのバー

### 1.6 つむぎからの提案
- 直近の記事のテーマから関連する3つの「次のお題」を生成
- リロードボタンで再生成（10回/日まで）
- 各提案をクリック → クイック生成パネルにテーマが流し込まれる

## 2. エディタ

### 2.1 ドキュメント構造 (Tiptap想定)
ブロック種別:
- `kicker` (固定1件、Display書体): 例「ESSAY · COFFEE & MORNINGS」
- `title` (h1相当): 必須
- `subtitle`: 任意
- `heading` (h2/h3): アクセントバー有無を選択可
- `paragraph`: 本文
- `image`: 画像 + キャプション
- `blockquote`: 引用
- `list` (順序付/順序なし)
- `code`: コードブロック
- `divider`: 罫線
- `ai-ghost`: AI補完候補（カーソル直後に薄字表示。Tabで accept）
- `ai-inline-suggestion`: 段落間に挿入される提案カード

### 2.2 ヘッダーバー
- 左: ← 戻る、タイトル(クリックで編集)、自動保存タイムスタンプ
- 右: 文字数進捗 (現在/目標)、プレビューボタン、**「noteに送る」ボタン**

### 2.3 文字数進捗
- 形式: `2,341 / 1,800字` (目標を超えても表示続行)
- 目標値はクイック生成時の設定から引き継ぐ。エディタ内で変更可
- 達成すると数値がアクセント色に + マスコットが `happy`

### 2.4 書式バー (Floating Format Bar)
- 文書上部中央に常駐、または選択範囲の上にフローティング表示
- 機能: H1 / H2 / 引用 / 太字 / 斜体 / 画像挿入 / タグ / **AIボタン**
- AIボタン: 選択範囲があれば「この部分を改善」、無ければサイドバーをトグル

### 2.5 ゴーストテキスト (インライン補完)
- ユーザーが2秒入力を止めると、続く1-2文をAIが提案、薄色で表示
- `Tab`: accept (本文に取り込み)
- `→`: 1単語ずつaccept
- `Esc` または任意入力: reject
- 設定で「常時オン / 段落終わりのみ / 手動 (`⌘+J`) / オフ」を切替

### 2.6 レイアウト切替 (本仕様の目玉)

#### 2.6.1 `editorLayout`
| 値 | 列構成 | キー |
|---|---|---|
| `two-pane` | アウトライン (220px) + エディタ | デフォルト |
| `focus` | エディタのみ (中央 12% 余白) | `⌘ + .` |
| `three-col` | アウトライン + エディタ + プレビュー(320px) | — |

#### 2.6.2 `aiPosition`
| 値 | 表示 | 説明 |
|---|---|---|
| `sidebar` | 右側 320px のチャットパネル | つむぎとの対話履歴を保持 |
| `inline` | 段落間に提案カード | 静的な提案、段階的に出現 |
| `floating` | 右下のバブル + マスコット | ドラッグ移動可、ピン留め可 |

両軸は独立。組み合わせは全9通り有効。設定はユーザー単位で永続化（`localStorage` + DB同期）。

### 2.7 アウトラインカラム
- 見出しブロックを階層表示（h2/h3）
- 各項目の状態ドット:
  - 灰 = `done`（書き終わった）
  - アクセント = `writing`（現在編集中）
  - ミント = `idea`（メモあり、未着手）
  - 薄灰 = `todo`（プレースホルダのみ）
- 「+」で新規見出し追加
- ドラッグで並び替え（本文も連動）
- 下部にAIからのアウトライン提案カード（見出しを増やす/分割する提案）

### 2.8 AIサイドバー (`aiPosition = sidebar`)
- ヘッダー: マスコット + 「つむぎ」+ モード表示 (聞き役 / 編集者 / 校正 / 構成)
- チャット履歴（記事ごとに保持）
- AIの返答内に「挿入」「書き直し」ボタン → エディタへ反映
- クイック指示チップ: もっと柔らかく / 具体例を3つ / 結びを書く / タイトルを5案
- 入力欄: 通常入力 + 「✦」で意図ボタン (要約 / 続きを書く / 校正 / トーン変換)

### 2.9 インライン提案 (`aiPosition = inline`)
- 段落の切れ目に「✦ つむぎの提案」カードが現れる
- 内容: 読者への問いかけ / 接続詞の置換 / 例示の追加 など
- アクション: **採用 / 別案 / 却下**
- 採用時はその場でブロック挿入、却下は二度と出さない

### 2.10 フロートAI (`aiPosition = floating`)
- 右下にマスコット + ふきだし
- 観察コメント（例: 「段落が3つ続いていて、息が詰まりそう」）
- マスコットクリックで全画面チャットに展開
- ドラッグで位置移動、4隅にスナップ
- ハードシャドウ (`4px 4px 0 #1B1A17`) でbluehamham風

### 2.11 画像挿入
- ドラッグ&ドロップ / 貼り付け / 「+」メニューから
- 自動アップロード（Vercel Blob / R2）
- AI代替テキスト自動生成（OpenAI Vision）
- キャプション編集可
- **AI画像生成 (オプション)**: ジャンルが「フォトエッセイ」等の時にプレースホルダから生成提案

### 2.12 タグ管理
- 記事末尾にタグチップ列
- 入力時に過去使ったタグをサジェスト
- ハッシュタグ形式に自動変換

## 3. プレビュー (note風 読み物表示)

### 3.1 表示
- 最大幅 720px、左右余白を取った1カラム
- フォントサイズと行間はnoteに合わせる（17px / 1.95）
- 見出しh2は左にアクセントバー（4px）
- 引用ブロックは左に3pxインクの罫線、斜体

### 3.2 メタ情報
- 著者アバター + 名前 + 投稿日 + 推定読了時間 (1分=400字)
- ヘッダー画像 (eyecatch)
- フォローボタン

### 3.3 インタラクション (モック)
- スキ (♡) + 数値、コメント数
- 公開ボタンで「noteに送る」モーダル

### 3.4 「noteに送る」モーダル
3つの送信経路を提示:
1. **Markdown ダウンロード** (.md ファイル)
2. **クリップボードへHTMLコピー** → note WYSIWYG に貼り付け推奨
3. **共有用プレビューURL作成** (`/p/[shareId]` で公開、編集不可)

公開予約 (日時指定) は別タブで設定。

## 5. 設定

### 5.1 プロフィール
- 名前、ID、プロフィール画像、bio、noteのプロフィールURL

### 5.2 マスコット設定
- 名前変更（デフォルト「つむぎ」）

### 5.3 執筆環境
- エディタレイアウト初期値

### 5.5 連携
- note連携状態（クリップボード / Markdown経由）
- データエクスポート (全記事JSON / Markdown zip)
- アカウント削除

## 6. モバイル固有の振る舞い

### 6.1 ホーム
- 挨拶カード + クイック生成入力 (1行)
- 統計3タイル横並び
- 下書きカードリスト (3件まで初期表示、スクロールで全件)
- 下部タブバー + 中央FAB

### 6.2 エディタ
- 集中モード前提（アウトラインは非表示、上のスライドメニューから呼び出し）
- 底に書式バー (横スクロール可)
- AI位置:
  - `sidebar` → 右側からスライドイン
  - `inline` → 段落間に出現
  - `floating` → 右下マスコット、タップで全画面

### 6.3 プレビュー
- ヘッダー画像が大きく出る note と同じ感覚
- 下部に固定アクションバー（スキ / コメント / 下書き戻る / 公開）

## 7. データモデル (PostgreSQL想定)

```sql
users (id, email, name, avatar_url, mascot_name, mascot_accent, created_at)

articles (
  id, user_id, title, subtitle, kicker,
  content_json,        -- Tiptap JSON
  content_text,        -- 全文検索用プレーンテキスト
  status,              -- draft | scheduled | published | archived
  goal_words, current_words,
  genre, tone, structure,
  cover_image_url,
  tags,                -- text[]
  share_id,            -- 共有URL用ランダムID
  published_at, scheduled_at,
  created_at, updated_at
)

ai_messages (
  id, article_id, role, -- user | assistant
  content, mode,        -- editor | reader | proofread | structure
  created_at
)

ai_suggestions (
  id, article_id, kind, -- ghost | inline | outline
  payload_json,
  status,               -- pending | accepted | rejected
  created_at
)

user_settings (user_id, editor_layout, ai_position, ghost_mode, ai_model, font_offset, ...)

writing_stats (user_id, date, words_written, articles_touched)
```

## 9. 状態遷移 (記事のステータス)

```
            ┌────────────────────────────────┐
            ↓                                 │
  [empty] → [draft] → [scheduled] → [published]
              │                       │
              ↓                       ↓
          [archived] ←──────────── [archived]
```

- `draft`: 作成直後、自由編集
- `scheduled`: 公開予約済（編集可、予約時刻にステータス遷移）
- `published`: 公開済 (note送信後、または共有URL発行後)
- `archived`: 一覧から除外、検索のみヒット

## 10. メトリクス (集計対象)

- 1日の執筆文字数

## 11. 非機能要件

- **Core Web Vitals**: LCP < 2.5s / CLS < 0.1 / INP < 200ms
- **i18n**: 日本語のみ (v1)。文字列は `/locales/ja.json` で集約
- **アクセシビリティ**: WCAG 2.1 AA 準拠
- **テスト**: Playwright で各画面のスモーク + AI出力の固定モック
- **ログ**: Sentry でエラー集約、PostHog で行動分析（オプトイン）
