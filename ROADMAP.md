# つむぐ（Tsumugu）ロードマップ

note クリエイターのための AI 共同執筆エディタ。  
本ドキュメントは開発の全フェーズと現在地を記録する。

---

## Phase 1 — スタイル設定 ✅

`/settings` ページ。自分の文体・構成スタイルを `StyleConfig` として登録・管理する。

**実装済み**

- `GET /api/style` — 保存済みスタイルの読み込み
- `PUT /api/style` — スタイルの保存
- `StyleDisplay` — 現在のスタイルを見やすく表示
- `StyleEditor` — フォーム式の編集UI（v2.0でJSON直編集からフォームに刷新）

---

## Phase 2 — スタイルAI解析 ✅

`/settings` ページ下部。既存のnote記事を貼り付けると、AIが文体を自動解析して `StyleConfig` を生成する。

**実装済み**

- `POST /api/style/analyze` — Anthropic tool_use でStyleConfigを構造化抽出
- `StyleAnalyzer` — テキスト貼り付け → 解析結果プレビュー → 適用 の3ステップUI

---

## Phase 3 — 記事生成 ✅

`/generate` ページ。書きたいテーマ・エピソードを入力し、Phase 1/2 で設定したスタイルに沿った記事をAIが生成する。

**実装済み**

- `GenerateForm` — 入力フォーム（テーマ、エピソード、ジャンル、口調、構成、長さ、ターゲット読者、追加メモ）
- `POST /api/generate` — StyleConfig を読んで Claude に渡し、タイトル候補 + Markdown本文を生成・保存
- `GET /api/articles` — 生成済み記事一覧
- `GET /api/articles/[id]` — 個別記事取得
- `PATCH /api/articles/[id]` — タイトル選択・ステータス更新
- ホーム画面の「DRAFTS · 進行中」に実際の記事一覧を表示

---

## Phase 4 — 整形・プレビュー・エディタ ✅

エディタとプレビュー、画像対応の基盤。

**実装済み**

- `/editor/[id]` — Tiptap 3 ベースの3カラムエディタ（OUTLINE / 本文 / AIつむぎ）
- `/preview/[id]` — note風レイアウトのプレビュー画面
- `/library` — 全記事一覧、進捗バー付き
- `POST /api/editor/assist` — 選択範囲ベースのAI支援（ブロックタイプ保持の置換）
- `POST /api/articles/[id]/format` — 本文にハッシュタグを付与してformattedTextを生成
- `POST /api/articles/[id]/header-image` — ヘッダー画像のアップロード
- `FormatPreview` — タイトル・本文コピーボタン付きプレビュー
- 「noteで開く」ボタン — クリップボードコピー + note.com の新規作成ページを開く

---

## Phase 5 — v2.0 デザイン刷新 ✅（2026年5月24日完了）

デザインシステムの全面刷新。「note の落ち着き + ポップで遊び心のあるアクセントを一滴 + AIは黒子」の3原則に基づき、デザイントークン・コンポーネント・全画面のトーンを統一。

**実装済み**

- デザイントークン（`@theme`）の刷新：ウォーム生成り背景、ピンクアクセント、Noto Serif JP / Noto Sans JP / Bebas Neue / JetBrains Mono のフォントスタック、ハードシャドウ
- マスコット「つむぎ」の実装：豆/しずく型のオリジナルSVG、5表情（idle / thinking / happy / wink / writing）
- UIプリミティブ：`Btn`、`Chip`、`Icon`、`BigNum`、`SectionHead`、`ImgPlaceholder`
- ホーム画面：STREAK / PUBLISHED / WORDS の数値表示、書きかけドラフト一覧、QUICK START、つむぎからの提案
- エディタ画面：OUTLINEパネル（状態ドット + NOWバッジ）、AIつむぎパネル（mood連動、黄マーカー強調）
- プレビュー画面：ヘッダー画像 + 著者行 + note風組版
- ライブラリ・設定・生成・インフォメーション画面：トークン統一、SectionHead適用

**開発記録**

- 推定時間 10〜16時間 → 実績約3時間（Claude Opus 4.7使用）
- スプリント1〜3に分割、各段階でコミット
- ブランチ戦略：`main`（v1完成版） → `v2.0-rebuild`（v2.0開発） → `main`にマージ

## Phase 6 — v3.0「READMEから自動入力」🚧（次に着手）

`/generate` ページに、READMEを貼り付けるだけで生成フォームの各フィールドが自動入力される機能を追加する。プラットフォーム（note / Qiita / Zenn）を選ぶことで、それぞれの文化に合った素材を抽出する。

**仕様書**：`docs/v3-planning.md`（完全な実装指示書）

**主な要素**

- 既存フォームを壊さない設計：上部に「READMEから自動入力」ボタンを追加し、解析結果でフォームを自動入力するだけ
- 3つのプラットフォームモード：note記事 / Qiita記事 / Zenn記事、それぞれシステムプロンプトを切り替え
- 入力方法：テキスト貼り付け、または `.md` / `.txt` ファイルのドラッグ&ドロップ
- 全件 Claude Haiku 要約パスを経由（精度優先）
- 文字数アラート：5000 / 10000 / 15000 字の3段階でユーザーに警告（商用利用を意識したコスト透明性）

**スプリント**

- スプリント1：API + 基本UI（推定2〜3時間）
- スプリント2：体験の磨き込み（推定1〜2時間）
